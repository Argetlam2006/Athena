# Athena

**AI-powered Football Decision Intelligence Platform**

> Transforming football data into confident, explainable decisions.

---

## What Is Athena?

Athena is a Football Decision Intelligence Platform that helps clubs transform large-scale football data into clear, explainable, and actionable decisions.

Rather than functioning as another analytics dashboard, Athena combines structured data, statistical analysis, machine learning, interactive visualization and explainable AI to support one objective:

> **Help decision-makers understand football better and make better football decisions.**

Every recommendation generated by Athena is traceable from a raw event all the way to a natural-language explanation. No step is skipped. No number is invented.

## The Athena Intelligence Framework

Every recommendation follows a strict 7-layer reasoning hierarchy:

```
Football Events       →  Raw observations (passes, shots, carries)
      ↓
Statistics            →  Aggregated per-90 metrics (xG, progressives, recoveries)
      ↓
Capabilities          →  8 composite ability scores
      ↓
Player Intelligence   →  Complete analytical profile
      ↓
Team Intelligence     →  Collective tactical identity
      ↓
Decision Intelligence →  Evidence-backed recommendations
      ↓
Natural Language      →  LLM explains existing analysis (never invents it)
```

### The 8 Core Capabilities

| # | Capability | Derived From |
|---|-----------|-------------|
| 1 | **Ball Progression** | Progressive passes, carries, deep completions, final-third entries |
| 2 | **Chance Creation** | Key passes, xA, shot assists, through balls, crosses |
| 3 | **Ball Security** | Pass completion, turnovers, dispossessions, miscontrols |
| 4 | **Press Resistance** | Successful dribbles, carries under pressure, retention rate |
| 5 | **Defensive Activity** | Pressures, recoveries, interceptions, blocks |
| 6 | **Attacking Threat** | xG, touches in box, shot quality, goals |
| 7 | **Physical Availability** | Minutes, starts, availability percentage |
| 8 | **Tactical Versatility** | Positions played, formation appearances, role adaptability |

> Capabilities intentionally avoid producing a single overall rating. Football decisions are context-dependent.

## Architecture

```
backend/
├── ingestion/          Data acquisition (StatsBomb Open Data)
├── etl/                ETL pipeline — clean, normalize, load to DuckDB
├── warehouse/          DuckDB warehouse + SQL views
├── analytics/          Feature engineering — per-90, percentiles, normalization
├── intelligence/       Athena Intelligence Framework — 8 capability scores
├── recommendation/     Decision engine — recruitment ranking, comparison
├── explanation/        AI explanation layer — prompts, scouting reports
├── api/                FastAPI route handlers
└── utils/              Logging, configuration

frontend/
├── app.py              Main Streamlit application (entry point)
├── pages/              One file per workspace
├── components/         Reusable UI components
└── charts/             Plotly chart builders

shared/
├── constants.py        Single source of truth for capabilities and metrics
├── schemas.py          Typed dataclasses for AIF pipeline contracts
└── models.py           Pydantic models for the API layer

sql/                    SQL schema, views, and example queries
data/
├── raw/                StatsBomb raw JSON files (not committed)
├── processed/          Cleaned Parquet files (not committed)
├── warehouse/          DuckDB database file (not committed)
└── sample/             Small sample for demonstration (CC BY-SA 4.0)
tests/                  Unit, integration, and end-to-end tests
docs/                   Architecture documents and decisions
```

## Quick Start

### Prerequisites

- Python 3.10+
- `make` (on Windows: Git Bash, WSL, or `choco install make`)

### Setup

```bash
# 1. Clone the repository
git clone https://github.com/bhaskarpareek/athena.git
cd athena

# 2. Create environment and install dependencies
make setup

# 3. Fetch sample data (StatsBomb Open Data, CC BY-SA 4.0)
make data

# 4. Validate data quality
make validate

# 5. Launch the platform
make app
```

The application will be available at `http://localhost:8501`.

### Manual Setup (without make)

```bash
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

pip install -r requirements.txt
pip install -e .

python -m backend.ingestion.load_data --sample
python -m backend.ingestion.validator

streamlit run frontend/app.py
```

## Project Philosophy

Athena is built on five principles:

1. **Decision Intelligence** — analytics that answer practical football questions
2. **Explainability** — every recommendation is transparent and auditable
3. **Evidence First** — visualizations and statistics come before AI
4. **Product Thinking** — every screen serves a specific football decision
5. **Human-Centered Intelligence** — augmenting analysts, not replacing them

## Architecture Decisions

Significant decisions are documented in [docs/ARCHITECTURE_DECISIONS.md](docs/ARCHITECTURE_DECISIONS.md).

Key decisions:
- DuckDB over PostgreSQL (ADR-001)
- Rule-based archetypes over K-Means (ADR-002)
- StatsBomb-only for Version 1 (ADR-003)
- Tactical Versatility as the 8th capability (ADR-004)
